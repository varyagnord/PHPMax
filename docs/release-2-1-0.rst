PyMax 2.1.0
===========

PyMax 2.1.0 расширяет библиотеку вокруг сценариев, которые раньше требовали
низкоуровневых вызовов: 2FA, заявки на вступление, QR-вход и запуск web
app-ботов. Ниже описаны изменения ветки ``dev/2.1.0`` относительно
стабильной версии ``2.0.1``.

Новые возможности
-----------------

* Добавлена работа с заявками на вступление в группы и каналы. Теперь можно
  получить список заявок через ``get_join_requests()``, подтвердить одну или
  несколько заявок через ``confirm_join_request()`` /
  ``confirm_join_requests()`` и отклонить их через ``decline_join_request()`` /
  ``decline_join_requests()``.
* Для заявок добавлены доменные типы ``Member`` и ``Presence``. Они описывают
  пользователя из списка заявок и его состояние присутствия.
* Управление 2FA стало полнее: появился ``check_2fa()`` для проверки, включен
  ли пароль 2FA у текущего аккаунта, и ``change_password()`` для смены
  текущего 2FA-пароля.
* Добавлен ``authorize_qr_login(qr_link)`` - метод для подтверждения входа по
  QR-ссылке.
* Добавлена поддержка начальных данных web app-ботов. Метод
  ``get_bot_init_data(bot_id, chat_id, start_param=None)`` возвращает
  ``InitData`` с ``query_id`` и ``url``.

Исправления
-----------

* Upload фото, видео и файлов теперь использует ``ExtraConfig.proxy`` и для
  прямой HTTP-отправки на upload URL, а не только для основного транспорта.
* Модель ``MaxApiError`` стала устойчивее к неполным ответам API: ``title`` и
  ``localized_message`` теперь могут отсутствовать в payload.

Изменения поведения
-------------------

* ``check_2fa()`` смотрит на ``profile_options`` текущего профиля. Если профиль
  еще не загружен или сервер не прислал эти признаки, метод возвращает
  ``False``.
* Если в ``ExtraConfig`` задан ``proxy``, upload-запросы тоже пойдут через
  него. Это полезно для единой сетевой настройки, но proxy должен поддерживать
  большие HTTP POST-запросы для файлов и видео.

Внутренние улучшения
--------------------

* Низкоуровневые enum-ы 2FA разделены по смыслу: ``ProfileOptions`` описывает
  признаки профиля, а ``TwoFactorAction`` - действия, которые отправляются в
  ``expectedCapabilities``.
* Внутренний ``ApiFacade`` получил ``bots``-сервис, а клиентские mixin-ы -
  методы для web app-ботов и заявок на вступление.
* Добавлен широкий pytest-набор для auth, chat/user/self/session API,
  сообщений, upload, runtime приложения, соединений, dispatch, протоколов,
  сессий, telemetry, файлов и bound-моделей.

Заметки для обновления
----------------------

* Для обычного кода на ``Client`` и ``WebClient`` обязательной миграции по
  diff не видно, deprecated API не добавлялись.
* Если вы импортировали низкоуровневый ``pymax.api.auth.enums.Capability``,
  замените его на ``TwoFactorAction`` для действий 2FA и на
  ``ProfileOptions`` для признаков профиля.
* Если код ожидал, что ``MaxApiError.title`` всегда строка, теперь нужно
  учитывать ``None``.
